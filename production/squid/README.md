# Squid Forward Proxy

Squid forward proxy with SSL bumping (MITM) for HTTPS traffic inspection and caching.

## Service Details

- **LoadBalancer IP**: 172.28.0.4
- **Port**: 3128
- **Protocol**: HTTP proxy with SSL bumping
- **Cache Size**: 128GB disk cache, 32GB RAM cache
- **Workers**: 8
- **Storage**: 131Gi PersistentVolumeClaim (openebs-hostpath)

## Features

- **SSL Bumping**: MITM for HTTPS traffic inspection using self-signed CA certificate
- **Optimized Caching**: Special refresh patterns for:
  - Docker registry layers (blobs, manifests, images)
  - APT packages (Packages, Sources, Release files)
- **Large Object Support**: Up to 32GB objects (for NVHPC runfiles, etc.)
- **Direct Routing**: All traffic goes directly to destination (no upstream proxy)

## Client Configuration

### 1. Install CA Certificate

To avoid SSL certificate warnings, install the squid CA certificate on client machines:

```bash
# Extract CA certificate from Kubernetes secret
kubectl get secret squid-ca -n squid -o jsonpath='{.data.tls\.crt}' | base64 -d > squid-ca.crt

# Install on Ubuntu/Debian
sudo cp squid-ca.crt /usr/local/share/ca-certificates/squid-ca.crt
sudo update-ca-certificates

# Install on RHEL/CentOS
sudo cp squid-ca.crt /etc/pki/ca-trust/source/anchors/
sudo update-ca-trust
```

### 2. Configure Proxy Settings

#### System-wide (Linux)

Add to `/etc/environment`:

```bash
http_proxy=http://172.28.0.4:3128
https_proxy=http://172.28.0.4:3128
HTTP_PROXY=http://172.28.0.4:3128
HTTPS_PROXY=http://172.28.0.4:3128
no_proxy=localhost,127.0.0.1,.zju.edu.cn
NO_PROXY=localhost,127.0.0.1,.zju.edu.cn
```

#### APT (Ubuntu/Debian)

Create `/etc/apt/apt.conf.d/02proxy`:

```
Acquire::http::Proxy "http://172.28.0.4:3128";
Acquire::https::Proxy "http://172.28.0.4:3128";
```

#### Docker

Add to `/etc/systemd/system/docker.service.d/http-proxy.conf`:

```ini
[Service]
Environment="HTTP_PROXY=http://172.28.0.4:3128"
Environment="HTTPS_PROXY=http://172.28.0.4:3128"
Environment="NO_PROXY=localhost,127.0.0.1,.zju.edu.cn"
```

Then reload and restart:

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

#### Git

```bash
git config --global http.proxy http://172.28.0.4:3128
git config --global https.proxy http://172.28.0.4:3128
```

```bash

  echo "Request $i:"

    -skI | grep -E "X-Cache"

Expected output:
```

X-Cache: MISS from squid
Request 2:

## Architecture

### SSL Bumping

Squid performs SSL bumping (MITM) using:

1. **CA Certificate**: Generated by cert-manager (`squid-ca` secret)
   - Self-signed CA certificate
   - Mounted into pod at `/etc/squid/bump.{crt,key}`

2. **SSL Certificate Database**: Dynamic certificate cache
   - Initialized by `security_file_certgen` in init container
   - Stored in emptyDir volume (ephemeral, regenerated on pod restart)
   - 20MB cache for generated certificates

### Cache Storage

- **Persistent Volume**: 131Gi PersistentVolumeClaim (openebs-hostpath)
- **Cache Type**: AUFS with 32 L1 dirs, 512 L2 dirs
- **Cache Location**: `/cache` in container
- **Memory Cache**: 32GB RAM cache
- **Deployment Strategy**: Recreate (required due to ReadWriteOnce PVC)

### Resource Allocation

- **Memory**:
  - Request: 34Gi (32GB cache_mem + 2GB overhead)
  - Limit: 40Gi (allows burst for large objects)
- **CPU**:
  - Request: 8 cores (matches 8 workers)
  - Limit: 16 cores (allows burst during high load)

## Troubleshooting

### Pod Fails to Start

Check init container logs:

```bash
kubectl logs -n squid deployment/squid -c init-squid
```

Common issues:
- Mirror download failures (mirrors.zju.edu.cn unreachable)
- ssl_db initialization errors

### Certificate Errors on Clients

1. Verify certificate is installed:
   ```bash
   openssl s_client -connect www.google.com:443 \
     -proxy 172.28.0.4:3128 2>&1 | grep -A2 "issuer"
   ```

2. Check certificate in Kubernetes:
   ```bash
   kubectl get certificate -n squid
   kubectl describe certificate squid-ca -n squid
   ```

### Cache Not Working

1. Check cache directory permissions:
   ```bash
   kubectl exec -n squid deployment/squid -- ls -la /cache
   ```

2. Check squid logs for cache errors:
   ```bash
   kubectl logs -n squid deployment/squid | grep -i cache
   ```

### Service Not Accessible

1. Check LoadBalancer IP assignment:
   ```bash
   kubectl get svc -n squid
   ```

2. Verify Cilium IPAM pool:
   ```bash
   kubectl get ciliumloadbalancerippool -n default
   ```

## Migration from Docker Compose

This deployment replaces the Docker Compose deployment at `/home/bowling/jenkins.clusters.zjusct.io/services/squid`.

Key differences:

| Aspect | Docker Compose | Kubernetes |
|--------|---------------|------------|
| IP Address | Host network | 172.28.0.4 (LoadBalancer) |
| Certificate | Manual (cert_gen.sh) | cert-manager (automatic) |
| ssl_db | Built into image | Init container (ephemeral) |
| Cache | Host path mount | PersistentVolumeClaim |
| Config | File mount | ConfigMap |
| Upstream proxy | 172.25.4.11:8081 | None (direct routing) |
